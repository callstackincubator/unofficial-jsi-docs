Detailed Description

Methods for starting and collecting instrumentation, an Instrumentation instance is associated with a particular Runtime instance, which it controls the instrumentation of. None of these functions should return newly created jsi values, nor should it modify the values of any jsi values in the heap (although GCs are fine).

using facebook::jsi::Instrumentation::HeapStatsUpdate = std::tuple<uint64_t, uint64_t, uint64_t>

A HeapStatsUpdate is a tuple of the fragment index, the number of objects in that fragment, and the number of bytes used by those objects. A "fragment" is a view of all objects allocated within a time slice.

◆ getRecordedGCStats()

virtual std::string facebook::jsi::Instrumentation::getRecordedGCStats ( )
pure virtual

Returns GC statistics as a JSON-encoded string, with an object containing "type" and "version" fields outermost. "type" is a string, unique to a particular implementation of jsi::Instrumentation, and "version" is a number to indicate any revision to that implementation and its output format.

Precondition
This call can only be made on the instrumentation instance of a runtime initialised to collect GC statistics.
Postcondition
All cumulative measurements mentioned in the output are accumulated across the entire lifetime of the Runtime.
Returns
the GC statistics collected so far, as a JSON-encoded string.
